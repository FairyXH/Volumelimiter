# Volumelimiter

Volumelimiter 是一个基于 **libxposed API 101** 的 LSPosed 模块，用于限制 Android 11 及以上系统的最大音量。

模块运行在 `system_server` 中，通过 Hook `AudioService` 处理硬件音量键、系统音量面板，以及应用通过 `AudioManager` 设置或调整音量的常见路径。它不修改 `/system`、`framework` 或 `services.jar`，配置通过 LSPosed Remote Preferences 实时同步。

> Volumelimiter 限制的是 Android 的离散音量档位，不会测量或保证真实声压级。耳机灵敏度、扬声器功率、蓝牙绝对音量、设备自身音量和厂商音效都会影响实际响度。

## 功能

- 设置作用于所有输出设备和应用的全局音量上限。
- 为扬声器、Bluetooth A2DP、有线耳机和 HDMI 设置独立上限。
- 为指定应用设置默认上限，并可进一步配置该应用的音量类别和输出设备上限。
- 通过“直接调用包名 → AudioFocus 所有者 → 前台任务”识别当前音频应用。
- 为媒体、铃声、通知、闹钟、系统声音、通话、Bluetooth SCO 和 Assistant 设置全局安全上限。
- 创建、编辑、复制和删除音量模板。
- 搜索全部已安装应用，跨搜索结果保持选择，并批量添加规则或应用模板。
- 新应用安装后可自动套用默认模板，并可选发送通知。
- 将全量配置导出为 JSON，或从 JSON 备份完整恢复。
- Debug 模式输出应用候选、命中规则、音量类别、输出设备、百分比和最终档位等决策信息。
- 所有 Hook、隐藏 API 和反射路径均采用保护性异常处理；无法兼容时优先放行，避免异常传播到 `system_server`。
- 设置页从 LSPosed 模块详情页打开，不创建桌面 Launcher 图标。

## 系统要求

- Android 11 或更高版本（API 30+）
- Root 环境
- 支持现代 libxposed API 101、`system_server` Hook 和 Remote Preferences 的 LSPosed
- 模块作用域：**System Framework / 系统框架**，静态作用域为 `system`
- 应用包名：`io.github.fairyxh.volumelimiter`

本项目按常见 AOSP `AudioService` 和 `MediaFocusControl` 方法设计，并对方法重载进行兼容查找，但不能保证所有厂商 ROM 的私有实现均完整生效。

## 安装与启用

1. 安装构建出的 APK。
2. 打开 LSPosed 管理器，在“模块”中找到 **Volumelimiter**。
3. 启用模块。
4. 确认作用域为 **System Framework / 系统框架**。无需勾选音乐、视频等普通应用。
5. 完整重启设备，使模块重新注入 `system_server`。
6. 从 LSPosed 的 Volumelimiter 模块详情页打开设置。
7. 开启“启用音量限制”，设置全局上限，再按需配置设备、应用、音量类别和模板。

首次安装、升级模块、修改作用域或重新启用模块后，应完整重启设备。仅重启播放器通常不会重新安装系统服务 Hook。

## 规则模型与优先级

### 基础覆盖关系

```text
命中的应用规则 > 当前输出设备规则 > 全局规则
```

1. 全局规则是所有场景的默认值。
2. 当前输出设备启用了独立规则时，设备值覆盖全局值，而不是与全局值取较小值。
3. 开启分应用限制且识别到已配置应用时，应用规则覆盖设备或全局规则。
4. 应用内部也采用“应用设备值覆盖应用默认值”的关系。
5. 应用音量类别上限和全局音量类别安全上限属于额外保护，最终与当前结果取较小值。
6. 关闭总限制开关后，所有设备、应用和音量类别限制停止生效。

可以概括为：

```text
基础上限 = 应用设备值（若有）
        或应用默认值（若命中应用）
        或全局设备值（若有）
        或全局默认值

最终上限 = min(基础上限, 应用音量类别上限, 全局音量类别安全上限)
```

未启用的可选上限不参与计算。

### 应用识别顺序

模块按以下顺序寻找音频应用：

1. AudioService 方法参数中直接携带的调用包名；
2. 当前 AudioFocus 所有者；
3. 系统返回的前台任务。

这使后台播放音乐时仍能应用播放器规则，而不只依赖屏幕上显示的 Activity。当应用放弃 AudioFocus、从焦点栈移除、Binder 死亡或被系统清理时，模块会同步移除缓存的所有者并回退到其他候选或全局规则。

> 如果应用退出界面后仍在后台播放并继续持有 AudioFocus，继续使用该应用规则是预期行为。停止播放、释放 AudioFocus 或结束进程后才会回退。

## 使用说明

### 设置全局上限

在“总配置”中：

1. 开启“启用音量限制”。
2. 设置默认百分比，例如 `60%`。
3. 所有没有更具体规则的设备和应用都会使用该值。

### 设置输出设备上限

在“输出设备覆盖”中找到目标设备：

1. 开启该设备的规则。
2. 设置独立限制值。
3. 连接该设备后，该值覆盖全局默认值。
4. 关闭设备规则即可恢复使用全局值。

当前单独识别：

| 设置项 | 识别范围 |
|---|---|
| 扬声器 | 内置扬声器 |
| 蓝牙耳机 | Bluetooth A2DP 输出 |
| 有线耳机 | 传统有线耳麦和有线耳机 |
| HDMI | HDMI 输出 |

USB 音频、BLE Audio、助听器、投屏设备及厂商私有路由可能无法单独识别，此时会回退到全局规则。多设备同时路由时，只能按模块识别出的一个设备类别应用规则。

### 添加和管理应用规则

1. 开启“启用分应用限制”。
2. 点击“搜索、选择与批量应用模板”。
3. 按应用名称或包名搜索。
4. 勾选一个或多个应用。选择状态在更换搜索词后仍会保留。
5. 点击“添加规则”，或选择一个模板批量应用。
6. 返回主页面，点击已配置应用进入详细编辑。

应用详细规则支持：

- 启用或停用该应用规则；
- 设置应用默认上限；
- 设置应用内各音量类别上限；
- 设置应用在不同输出设备上的上限；
- 删除该应用规则。

应用管理读取全部已安装包，不要求应用具有 Launcher 图标。主页面只预览前 20 条已配置规则；其他规则可在应用管理中搜索，不会被删除。

### 设置音量类别安全上限

全局支持以下音量类别：

- 媒体
- 铃声
- 通知
- 闹钟
- 系统声音
- 通话
- Bluetooth SCO
- Assistant

例如设备上限为 `80%`，媒体安全上限为 `50%`，则媒体最终不高于 `50%`；未启用类别安全上限的闹钟仍按设备或全局规则计算。

### 创建和使用模板

模板包含：

- 应用默认上限；
- 应用音量类别上限；
- 应用输出设备上限。

使用步骤：

1. 在“模板”区域点击“创建模板”。
2. 输入名称并配置需要的上限。
3. 保存后，可编辑、复制或删除模板。
4. 在应用管理中选择多个应用，点击“批量应用模板”。
5. 选择模板后，所有选中应用会获得同一组规则。

复制模板适合从已有配置创建轻微不同的方案。删除模板不会自动删除已经应用到各应用上的规则。

### 自动配置新安装应用

1. 先创建至少一个模板。
2. 在“新安装应用行为”中选择默认模板。
3. 开启“新应用自动套用模板”。
4. 可选开启“自动配置后发送通知”。Android 13 及以上需要授予通知权限。

`system_server` 中的模块代码动态监听 `PACKAGE_ADDED`，仅处理 `EXTRA_REPLACING=false` 的真正新增事件，并直接把默认模板复制为应用独立规则。该路径不依赖 Volumelimiter App 进程存活，也不会在首次启用模块时扫描并配置全部存量应用。已有规则（包括卸载后重装留下的规则）不会被覆盖。

启用通知后，由 system_server 的系统 Context 创建通知渠道并发送通知，因此通知归属为 Android 系统；点击通知会打开对应应用的规则编辑页。

### 导出和导入配置

“配置导入导出”支持 JSON 全量备份。

导出内容包括全局设置、设备规则、应用规则、模板和新应用行为设置。导入前会读取并校验完整 JSON，确认后一次性覆盖现有配置；格式错误或写入失败时会报告错误，而不是导入一部分配置。

建议在批量调整规则、升级模块或更换 ROM 前导出备份。备份文件可能包含已安装应用包名，分享前请检查隐私信息。

### 重置全部设置

页面底部的“重置全部设置”会清除：

- 全局和设备限制；
- 所有应用规则；
- 所有音量类别安全上限；
- 所有模板；
- 新应用自动配置选项；
- Debug 日志选项。

此操作不会卸载模块。

## 上限档位计算

Android 音量通常由离散档位表示。模块根据系统报告的最大档位计算限制档位：

```text
限制档位 = ceil(系统最大档位 × 最终百分比 ÷ 100)
```

规则如下：

- `0%` 对应 0 档；
- 非零百分比至少保留 1 档；
- 结果不会超过系统最大档位；
- 使用向上取整，避免低百分比在档位较少的音量类别上意外变成 0。

例如媒体音量最大为 15 档：

| 百分比 | 计算 | 限制档位 |
|---:|---:|---:|
| 20% | `ceil(15 × 20 / 100)` | 3 |
| 40% | `ceil(15 × 40 / 100)` | 6 |
| 70% | `ceil(15 × 70 / 100)` | 11 |
| 80% | `ceil(15 × 80 / 100)` | 12 |

硬件音量键使用系统当前活动音量类别进行判断。系统传入默认建议流时，模块会通过 `getActiveStreamType()` 解析，不会把 flags、uid 等后续参数误认为音量类别。

## 示例

### 示例一：全局、设备与应用覆盖

配置：

- 全局上限：`60%`
- 蓝牙 A2DP：`80%`
- 视频应用 A 默认上限：`40%`
- 媒体安全上限：未启用

结果：

| 场景 | 生效上限 | 原因 |
|---|---:|---|
| 扬声器播放普通应用 | 60% | 使用全局规则 |
| 蓝牙播放普通应用 | 80% | 设备规则覆盖全局规则 |
| 蓝牙播放视频应用 A | 40% | 应用规则覆盖设备规则 |
| 退出并结束视频应用 A | 80% | 应用规则释放后回退到蓝牙规则 |

如果再设置全局媒体安全上限为 `30%`，以上所有媒体场景最终都不高于 `30%`。

### 示例二：应用按设备设置不同上限

配置：

- 全局上限：`70%`
- 音乐应用 B 默认上限：`50%`
- 音乐应用 B 的扬声器上限：`35%`
- 音乐应用 B 的蓝牙上限：`65%`

结果：

- 音乐应用 B 使用扬声器时为 `35%`；
- 音乐应用 B 使用蓝牙时为 `65%`；
- 音乐应用 B 使用未单独配置的设备时为 `50%`；
- 其他应用仍使用全局或全局设备规则。

### 示例三：儿童平板

- 全局上限：`50%`
- 蓝牙上限：`40%`
- 媒体安全上限：`50%`
- 为视频、游戏应用批量套用“儿童模式”模板：应用默认 `35%`

普通外放不超过约一半档位，蓝牙进一步降低，选中的视频和游戏应用使用更严格的 `35%`。

### 示例四：车载蓝牙

- 全局上限：`55%`
- Bluetooth A2DP 上限：`85%`
- 媒体安全上限：不启用

普通外放保持较低上限，连接车载 A2DP 后允许使用更高档位。设备规则属于覆盖，因此可以高于全局默认值。

### 示例五：后台音乐播放

- 音乐应用 C：`45%`
- 浏览器：无应用规则

音乐应用 C 开始播放并获得 AudioFocus 后，即使切到浏览器，媒体音量仍可使用音乐应用 C 的 `45%` 规则。停止播放并释放 AudioFocus 后，模块回退到浏览器候选、当前设备或全局规则。

## 验证是否生效

1. 启用模块、确认作用域并完整重启设备。
2. 设置一个容易观察的全局上限，例如 `40%`。
3. 播放媒体并连续按硬件音量加键，确认无法超过预期档位，同时仍可在限制范围内升降。
4. 打开系统音量面板，确认拖动滑块也不能超过上限。
5. 切换扬声器、蓝牙和有线耳机，分别验证设备规则。
6. 为某应用设置明显不同的上限，在应用内按音量键验证规则。
7. 停止播放并从多任务划掉应用，确认上限回退到设备或全局值。
8. 测试后台播放，确认仍持有 AudioFocus 时继续应用播放器规则。
9. 在 LSPosed 日志中筛选 `Volumelimiter`，确认存在 `Installed N AudioService hooks` 且 `N > 0` 的记录。

修改规则后，如果当前音量已经高于新上限，模块不会主动制造一次音量事件立即降低它；下一次通过硬件按键、系统面板或应用接口调整音量时才会执行限制。

## Debug 日志

仅在排查兼容性问题时开启 Debug。决策日志可能包含：

- 音量类别和输出设备；
- 直接调用应用；
- AudioFocus 应用；
- 前台应用；
- 最终选中的应用规则；
- 规则是否命中；
- 最终百分比；
- 限制档位和系统最大档位。

典型字段类似：

```text
LimitDecision stream=3 device=speaker directApp=...
focusApp=... foregroundApp=... selectedApp=...
appRuleMatched=true percent=20 maximum=3/15
```

提交日志前，请删除账号、设备标识、蓝牙地址及其他隐私信息。

## 兼容性与已知限制

- 当前 Hook 查找 `setStreamVolume`、`setStreamVolumeWithAttribution`、`adjustStreamVolume`、`adjustSuggestedStreamVolume` 和 `adjustSuggestedStreamVolumeForUid` 的兼容重载。
- 若 ROM 改名、彻底更改参数布局、绕过 `AudioService` 或阻止隐藏 API 访问，功能可能部分或全部失效。
- Hook 安装、路由识别、AudioFocus 跟踪和前台应用查询均采用失败放行策略；系统稳定性优先于强制限制。
- AudioFocus 是音频所有权近似值。应用错误持有焦点或厂商修改焦点栈行为时，应用归属可能不准确。
- 多窗口、分屏和画中画场景下，前台任务取决于系统返回结果。
- Bluetooth SCO 是音量类别，不等同于 Bluetooth A2DP 输出设备分类。
- BLE Audio、USB 音频、助听器、投屏和厂商私有输出通常回退到全局规则。
- 蓝牙绝对音量和耳机自身音量可能形成另一套音量控制，实际效果依设备而异。
- `50%` 档位不代表一半声压或一半主观响度。
- Release 构建当前使用 Debug 签名，适合测试和自行构建，不是正式发布签名。

## 故障排查

### 没有桌面图标

这是预期行为。请从 LSPosed 管理器的模块详情页进入设置。

### 设置页无法连接 LSPosed 服务

确认：

1. 当前 LSPosed 支持 libxposed API 101 Remote Preferences；
2. 模块已启用；
3. 从 LSPosed 模块详情页启动设置；
4. 启用或升级模块后已经重启设备。

### 设置后没有效果

依次检查：

1. Android 版本为 11 或以上；
2. LSPosed 和模块均已启用；
3. 作用域为 System Framework / `system`；
4. 修改作用域或启用模块后已完整重启；
5. 总限制开关已开启；
6. 当前输出设备、音量类别和应用与配置相符；
7. LSPosed 日志中的 AudioService Hook 数量大于 0。

### 应用规则不生效

- 确认“启用分应用限制”已开启；
- 确认应用规则自身已启用；
- 开启 Debug，检查 `directApp`、`focusApp`、`foregroundApp`、`selectedApp` 和 `appRuleMatched`；
- 播放器在后台播放时，应关注 AudioFocus 候选，而不是只看前台 Activity；
- 某些 ROM 可能修改 AudioFocus 或前台任务隐藏 API，此时模块会回退到其他候选、设备或全局规则。

### 应用退出后仍使用应用上限

1. 确认应用是否仍在后台播放；
2. 确认通知栏中是否仍有媒体会话；
3. 停止播放并从多任务划掉应用；
4. 开启 Debug，检查是否收到 `AudioFocus release via=...` 日志；
5. 若应用已经结束但 `focusApp` 仍长期指向旧包名，请提交 ROM、LSPosed 版本和相关日志。

### 硬件音量键不能正常调整

- 确认安装的是包含建议流解析修复的新版本，并已重启设备；
- 查看 Debug 日志中的 `stream`、`percent` 和 `maximum`；
- 例如媒体最大 15 档、限制 20% 时，应显示 `maximum=3/15`；
- 对比硬件音量键和系统音量面板是否表现一致，并在反馈中说明差异。

### 系统异常或无声

1. 在 LSPosed 中停用模块；
2. 重启设备；
3. 确认停用后问题是否消失；
4. 提交设备型号、Android/ROM/LSPosed 版本、配置、复现步骤和日志。
