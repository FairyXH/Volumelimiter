# Volumelimiter

Volumelimiter 是一个基于 **libxposed API 101** 的 LSPosed 模块，用于限制 Android 11 及以上系统的最大音量。模块运行在 `system_server` 中，通过 Hook `AudioService` 处理音量键、系统音量面板和应用调用 `AudioManager` 设置音量的常见路径，不修改 `/system`、`framework` 或 `services.jar`。

> 本模块限制的是 Android 的离散音量档位，并不能测量或保证真实声压级。不同耳机、扬声器、蓝牙绝对音量和厂商音效会显著影响实际响度。

## 功能

- 一个作用于所有输出设备的**总限制开关和限制值**。
- 扬声器、蓝牙 A2DP、有线耳机、HDMI 各自拥有独立开关和限制值。
- 可选的**分应用限制**：指定应用位于前台时使用该应用的上限。
- 可选的音量类别安全上限：媒体、铃声、通知、闹钟、系统声音、通话、Bluetooth SCO、Assistant。
- 一键重置所有配置。
- 配置通过 LSPosed Remote Preferences 实时同步给 `system_server`。
- 设置页仅从 LSPosed 模块详情页打开，不创建桌面 Launcher 图标。
- Hook 与隐藏 API 查询采用保护性异常处理；不兼容时跳过或回退，不让模块异常传播到 `system_server`。

## 配置优先级

输出设备和应用配置采用类似“总轨 / 分轨”的覆盖关系：

```text
前台应用配置 > 当前输出设备配置 > 总配置
```

1. **总配置**默认作用于所有输出设备。
2. 当前设备启用了自己的配置时，设备值**覆盖**总配置，而不是与总配置取较小值。
3. 开启“分应用限制”后，若当前前台应用存在已启用规则，应用值再覆盖设备或总配置。
4. “音量类别安全上限”是额外保护。若启用某个音量类别，最终结果会与该类别上限取较小值。
5. 关闭总限制开关时，所有设备、应用和音量类别限制均停止生效。

例如：

- 总配置：`60%`
- 扬声器覆盖：未开启
- 蓝牙耳机覆盖：`80%`
- 分应用总开关：开启
- 某视频应用：`40%`
- 媒体类别安全上限：未开启

那么：

| 当前场景 | 生效上限 |
|---|---:|
| 扬声器播放普通应用 | 60%（总配置） |
| 蓝牙耳机播放普通应用 | 80%（设备覆盖总配置） |
| 蓝牙耳机中前台运行该视频应用 | 40%（应用覆盖设备配置） |

如果再启用“媒体类别安全上限 = `30%`”，上述三种媒体播放场景最终都会被压到不高于 `30%`。

## 支持的配置

### 输出设备

| 设置项 | 当前识别范围 |
|---|---|
| 扬声器 | 内置扬声器 |
| 蓝牙耳机 | Bluetooth A2DP 输出 |
| 有线耳机 | 传统有线耳麦和有线耳机 |
| HDMI | HDMI 输出 |

USB 音频、BLE Audio、助听器、投屏设备及厂商私有路由可能无法单独识别；这类设备会回退到总配置。多设备同时路由时也只能按模块识别出的一个类别应用规则。

### 音量类别安全上限

- 媒体
- 铃声
- 通知
- 闹钟
- 系统声音
- 通话
- Bluetooth SCO
- Assistant

音量类别限制是可选的额外安全帽，不影响“前台应用 > 设备 > 总配置”的覆盖顺序。

## 系统要求

- Android 11 或更高版本（API 30+）
- Root 环境
- 支持现代 libxposed API 101、`system_server` Hook 与 Remote Preferences 的 LSPosed
- 模块作用域：System Framework，对应静态作用域 `system`
- 应用包名：`io.github.fairyxh.volumelimiter`

本项目按 Android 11+ 设计，并针对常见 AOSP `AudioService` 方法进行兼容查找，但不能保证所有 Android 版本和所有厂商 ROM 均完整生效。

## 安装与启用

1. 安装构建出的 APK。
2. 打开 LSPosed 管理器，在“模块”中找到 **Volumelimiter**。
3. 启用模块。
4. 确认作用域为 **System Framework / 系统框架**（内部静态作用域为 `system`）。无需勾选音乐、视频等普通应用。
5. 完整重启设备，让模块重新注入 `system_server`。
6. 从 LSPosed 的 Volumelimiter 模块详情页打开设置。
7. 开启“启用音量限制”，设置总上限，再按需启用设备或应用覆盖。

首次安装、升级模块、修改作用域或重新启用模块后，应完整重启设备。仅重启播放器通常不会重新安装系统服务 Hook。

## 使用说明

### 1. 设置所有设备的默认上限

在“总配置”中：

1. 开启“启用音量限制”。
2. 将滑块设置为所需百分比，例如 `60%`。
3. 所有未配置独立覆盖的输出设备都会使用该值。

### 2. 为某个输出设备单独设置

在“输出设备覆盖”中找到目标设备：

1. 开启该设备的开关。
2. 设置设备自己的限制值。
3. 连接该设备后，其值会覆盖总配置。
4. 关闭设备开关即可恢复使用总配置。

### 3. 添加前台应用规则

1. 打开“启用分应用限制”总开关。
2. 点击“添加应用规则”。
3. 从具有 Launcher 入口的已安装应用列表中选择应用。
4. 设置该应用的限制值。
5. 该应用位于前台且系统能够识别时，应用配置覆盖设备或总配置。
6. 使用对应的“删除应用规则”按钮可移除规则。

分应用识别通过 `system_server` 内部任务信息进行最佳努力查询。厂商修改相关隐藏 API、分屏/画中画、多窗口、没有 Launcher 入口的应用或前台任务信息不可用时，可能无法应用对应规则；此时模块安全回退到设备或总配置。

### 4. 设置音量类别安全上限

如果希望无论设备或应用如何覆盖，某类声音都不超过固定值，可在“音量类别安全上限”中启用对应类别。

例如：设备覆盖为 `80%`，但媒体类别安全上限为 `50%`，则媒体最终上限为 `50%`；闹钟若未启用类别限制，则仍按设备或总配置计算。

### 5. 重置设置

在页面底部点击“重置全部设置”，确认后会：

- 关闭总限制和分应用限制；
- 清除所有设备覆盖；
- 清除所有应用规则；
- 清除所有音量类别安全上限；
- 关闭 Debug 日志；
- 将未配置的百分比恢复为默认 `100%`。

## 上限计算与详细效果示例

模块使用系统报告的最大音量档位计算整数上限：

```text
限制档位 = floor(系统最大档位 × 最终百分比 ÷ 100)
```

即整数除法向下取整。

### 详细示例

假设手机媒体音量最大档位为 `15`，配置如下：

- 总限制：已开启，`70%`
- 扬声器：未启用独立配置
- 蓝牙耳机：已启用，`80%`
- 分应用限制：已开启
- 视频应用 A：已启用，`40%`
- 媒体类别安全上限：未启用

结果：

1. **扬声器 + 普通音乐应用**
   - 没有扬声器覆盖，也没有应用规则。
   - 使用总限制 `70%`。
   - `floor(15 × 70 / 100) = 10`。
   - 按音量加键或应用请求设置到 15 时，允许的最高 index 为 10。

2. **蓝牙耳机 + 普通音乐应用**
   - 蓝牙设备配置覆盖总配置，使用 `80%`。
   - `floor(15 × 80 / 100) = 12`。
   - 最高 index 为 12。

3. **蓝牙耳机 + 前台视频应用 A**
   - 应用规则优先级最高，使用 `40%`。
   - `floor(15 × 40 / 100) = 6`。
   - 最高 index 为 6。

4. **切回没有应用规则的应用**
   - 前台应用规则不再命中。
   - 重新回退到蓝牙设备的 `80%`，最高 index 为 12。

如果设置生效前当前音量已经高于新上限，模块不会主动制造一次音量变更把它立刻拉低；下一次按音量加、使用音量面板或应用调用常规设置音量接口时才会执行限制。建议修改规则后主动调整一次音量进行验证。

## 常见使用场景

### 儿童使用平板

- 总限制：`50%`
- 扬声器：不单独覆盖
- 蓝牙：`40%`
- 媒体类别安全上限：`50%`

这样日常外放默认不超过一半档位，连接蓝牙耳机时进一步降低。

### 车载蓝牙需要更高音量

- 总限制：`55%`
- 蓝牙：`85%`
- 媒体类别安全上限：不启用

普通外放使用较低总上限，连接车载 A2DP 后允许使用更高档位。设备覆盖是替代总配置，因此蓝牙可高于总限制。

### 只限制某个容易突然变响的视频应用

- 总限制：`75%`
- 分应用限制：开启
- 视频应用：`35%`
- 其他设备覆盖：按需设置

视频应用位于前台时使用 `35%`，退出后恢复设备或总配置。

### 保证闹钟不被媒体规则影响

- 总限制：`70%`
- 媒体类别安全上限：`50%`
- 闹钟类别安全上限：不启用

媒体最高为 `50%`，闹钟仍按总配置或设备覆盖计算。修改提醒类音量后务必进行一次实际闹钟测试。

## 验证是否生效

1. 开启模块、设置作用域并完整重启。
2. 设置一个容易观察的总上限，例如 `40%`。
3. 播放媒体并连续按音量加键，确认无法继续超过预期档位。
4. 打开系统音量面板尝试拖到最大。
5. 切换扬声器、蓝牙和有线耳机，分别验证设备覆盖。
6. 开启分应用总开关并添加一个差异明显的规则，切换前台应用验证回退和覆盖。
7. 在 LSPosed 日志中筛选 `Volumelimiter`，确认存在类似 `Installed N AudioService hooks` 且 `N > 0` 的记录。

## 兼容性与已知限制

- 当前 Hook 固定查找 `setStreamVolume`、`setStreamVolumeWithAttribution`、`adjustStreamVolume`、`adjustSuggestedStreamVolume` 和 `adjustSuggestedStreamVolumeForUid` 的兼容重载。
- 若 ROM 改名、彻底更改参数布局、绕过 `AudioService` 或阻止隐藏 API 访问，功能可能部分或全部失效。
- Hook 安装、设备路由和前台应用识别均采取失败放行策略；系统稳定性优先于强制限制。
- 蓝牙设备项当前主要识别 A2DP。Bluetooth SCO 是独立音量类别，不等同于完整的蓝牙设备分类。
- BLE Audio、USB 音频、助听器、投屏和厂商私有输出通常回退到总配置。
- 多窗口、分屏和画中画场景下，“前台应用”可能取决于系统返回的顶部任务。
- 蓝牙绝对音量、耳机自身音量和厂商音效会改变实际听感。
- `50%` 档位不代表一半声压或一半主观响度。
- Release 构建当前使用 Debug 签名，适合测试和自行构建，不是正式发布签名。

## 故障排查

### 没有桌面图标

这是预期行为。请从 LSPosed 管理器的模块详情页进入设置。

### 设置页一直显示 LSPosed 服务未连接

确认模块已由支持 libxposed API 101 Remote Preferences 的 LSPosed 识别，并从 LSPosed 模块详情页启动设置。必要时启用模块并重启设备。

### 设置后没有任何效果

依次确认：

1. Android 版本为 11 或以上。
2. LSPosed 和模块均已启用。
3. 作用域为 System Framework / `system`。
4. 启用模块或修改作用域后已完整重启。
5. 总限制开关已开启。
6. 测试的设备和前台应用与规则相符。
7. LSPosed 日志中 Hook 数量大于 0。

### 分应用规则不生效

- 确认“启用分应用限制”总开关已打开。
- 确认应用规则自身开关已打开。
- 尝试让目标应用全屏位于最前台。
- 某些 ROM 或多窗口模式无法可靠返回顶部任务；模块会回退设备或总配置。

### 系统异常或无声

1. 在 LSPosed 中停用模块。
2. 重启设备。
3. 确认停用后问题是否消失。
4. 提交设备型号、Android/ROM/LSPosed 版本、配置、复现步骤和 LSPosed 日志。
